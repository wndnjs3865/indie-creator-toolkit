Defold is King's open-source 2D specialist optimized for mobile and HTML5. The right pick depends on platform target, scope, and how you feel about Unity post-2023. Here's the honest call.

## How to choose

- **Pick [Unity](https://unity.com)** if you're shipping 3D commercial games, mobile is your monetization target, or you specifically need the asset store ecosystem.
- **Pick [Defold](https://defold.com)** if you're shipping 2D to mobile or HTML5, binary size matters, and the Lua scripting model fits your team.

## The math

Unity Personal is free up to $200k/yr revenue, then $25/user/mo for Pro. Defold is $0 at every scale, forever, no asterisks.

For most indie projects, neither tool's licensing matters in year one (you're not at $200k). The decision is platform fit: Unity for 3D commercial / mobile-at-scale, Defold for 2D mobile / HTML5 specifically.

## Verdict

| Situation | Pick |
|---|---|
| 3D commercial / AAA-aspiring | **Unity** |
| Mobile-first commercial | **Unity** |
| 2D mobile-first | **Defold** |
| HTML5 / web games | **Defold** |
| Need the massive asset store | **Unity** |
| Care about open-source | **Defold** |
| Building portfolio for studio hiring | **Unity** (still industry standard) |
| Already comfortable with Lua | **Defold** |

If you're between the two — and your project is 2D — try Defold first. The binary-size advantage and zero-royalty model are dominant for 2D indie work. Unity remains the right answer specifically for 3D, mobile-at-scale, or asset-store-dependent projects.
