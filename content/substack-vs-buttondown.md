You're picking between [Substack](https://substack.com) and [Buttondown](https://buttondown.com). Both can run a paid newsletter in 2026 — but they speak to different writers. Substack is built for the network effect; Buttondown is built for indie devs and markdown loyalists. Here's the honest call.

## How to choose

- **Pick [Substack](https://substack.com)** if you're starting from zero, you write for a general audience, and you want the platform to help you find readers.
- **Pick [Buttondown](https://buttondown.com)** if you write technical content, you live in markdown editors, and you'd rather pay $9/mo than 10% of paid-sub revenue forever.

## The math

At 1,000 paid subs × $5/mo, Substack takes $500/mo. Buttondown at the equivalent tier costs ~$50-90/mo flat. Past about ~150 paid subs at $5, Buttondown is dramatically cheaper.

The trade-off: Substack delivers readers via recommendations. Buttondown expects you to bring an audience. If you're a technical writer with an existing GitHub/Twitter following to import, Buttondown's economics dominate. If you have no audience and need help getting one, Substack's network is worth the 10%.

## Verdict

| Situation | Pick |
|---|---|
| Starting from zero, general audience | **Substack** |
| Technical / indie-dev audience | **Buttondown** |
| You write in markdown editors | **Buttondown** |
| You want recommendations to grow you | **Substack** |
| You'd rather pay flat than revenue share | **Buttondown** |
| Need automation / sequences | Neither — use [Kit](https://kit.com) |

If you're a technical writer leaning toward Substack for the audience help — try Buttondown first with your existing GitHub/Twitter network. The migration tax later (if you outgrow Buttondown) is real.
