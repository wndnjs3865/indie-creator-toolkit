You're picking between [Notion](https://notion.so) and [Logseq](https://logseq.com). Both have generous free tiers. Both can hold notes, tasks, and research. But they pull in opposite directions: Notion is cloud-hosted, database-driven, and team-friendly. Logseq is local-first, outliner-style, and privacy-focused. Here's how to pick.

## How to choose

- **Pick [Notion](https://notion.so)** if you collaborate, you want a database/wiki layer, or you embed lots of media. Cloud-hosting is a feature when you have collaborators.
- **Pick [Logseq](https://logseq.com)** if you work alone on research, you want local files you own, and the outliner workflow fits how you think.

## The math

Logseq is $0, forever, no asterisks — so the cost comparison is moot. The interesting comparison is "what does Notion's $14/mo (or free, if solo) buy that Logseq doesn't have?"

The honest answer: collaboration, database views, embed-rich blocks, and managed sync. If those matter, Notion is worth the cost. If they don't, Logseq's free + local-first is dominant.

## Verdict

| Situation | Pick |
|---|---|
| Solo personal research / journaling | **Logseq** |
| Want local files + privacy | **Logseq** |
| Collaborating with anyone | **Notion** |
| Need database views (calendar, kanban) | **Notion** |
| Care about open-source insurance | **Logseq** |
| Want outliner UX (Roam-style) | **Logseq** |
| Embed lots of media (videos, files) | **Notion** |

If you're between the two — try Logseq first if you're solo (it's free and privacy-respecting). If by month two you're missing collaboration or database views, Notion is the move. The reverse migration (Notion → Logseq) is harder once you've built Notion-specific databases.
